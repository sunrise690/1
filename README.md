# 本地刷题系统

这是一个纯前端本地刷题网页，使用 `HTML + CSS + JavaScript` 实现，不需要后端和数据库。

## 项目结构

- `index.html`：页面入口
- `style.css`：界面样式
- `app.js`：题库解析、练习逻辑、localStorage 保存逻辑
- `questions.txt`：题库文本
- `README.md`：使用说明

## 如何运行

方式一：直接双击 `index.html` 打开。

如果浏览器阻止网页自动读取本地 `questions.txt`，点击右上角“导入 txt”，选择 `questions.txt` 即可。

方式二：平板离线使用。

直接把 `tablet-offline.html` 发送到平板，例如通过微信、QQ、邮箱、网盘、数据线或 AirDrop。平板收到后用浏览器打开这个 HTML 文件即可，不需要和电脑连接同一个 Wi-Fi。

方式三：使用本地静态服务自动读取 `questions.txt`。

在当前目录打开命令行，运行：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 如何替换题库

把完整题库内容粘贴到 `questions.txt` 中并保存。建议使用 UTF-8 编码。

支持的基础格式如下：

```text
一、单选题（共193题）
题干
A. 选项
B. 选项
C. 选项
D. 选项
正确答案: C

二、多选题（共151题）
题干
A. 选项
B. 选项
C. 选项
D. 选项
正确答案: A B C D
```

也支持题干前带题号，例如：

```text
1. 这是一道题干
A. 选项
B. 选项
正确答案: A
```

多选答案可以写成 `A B C`、`ABC` 或 `A、B、C`。判题时必须完全一致，少选或多选都算错。

也兼容部分平台导出的换行格式，例如：

```text
A
.
选项内容
```

题目中夹杂的 `单选题 (0 分) 0分`、`多选题 (0 分) 0分`、`此题未答` 等导出痕迹会自动忽略。

## 如何导入完整题库

1. 打开 `index.html`。
2. 点击右上角“导入 txt”。
3. 选择你的完整题库 txt 文件。
4. 页面会自动解析并开始练习。

导入后的题库文本会缓存在浏览器 localStorage 中。下次直接打开页面时，如果浏览器仍不能自动读取 `questions.txt`，会优先使用上次导入的题库。

如果已经导入过自己的 txt，之后重新打开网页会优先恢复这份导入题库，并继续保留答题记录、错题和收藏。想回到项目默认题库时，点击页面里的“恢复默认题库”。

## 已保存的数据

系统会使用 localStorage 保存：

- 错题
- 收藏题
- 答题记录
- 当前进度

点击页面里的“清空记录”会清除答题记录和错题本，但不会清除收藏题。
